# Vshokolade — обновлённый сайт

## Что сделано

- исправлен сценарий подтверждения заказа на телефоне: после отправки формы сайт открывает WhatsApp через мобильный deep-link и оставляет резервную зелёную кнопку подтверждения;
- исправлена ошибка в success-модалке (раньше скрипт обращался к отсутствующему элементу и ломал часть сценария);
- кнопка «Выбрать подарок» теперь переводит пользователя прямо к каталогу;
- добавено 2 языка: русский и казахский;
- каталог вынесен в отдельный файл данных `data/products.tsv`;
- добавена поддержка внешней таблицы: если указать опубликованную ссылку на Google Sheets/TSV/CSV в `js/catalog-source.js`, каталог будет подгружаться из таблицы автоматически;
- все карточки каталога, спецнаборов и Dubai Chocolate теперь рендерятся из файла данных, а не зашиты в HTML.

## Главные файлы

- `index.html` — структура страницы;
- `css/style.css` — стили;
- `js/app.js` — логика сайта, языки, корзина, WhatsApp, Bitrix24;
- `js/site-content.js` — тексты интерфейса и блок «Инфо» на двух языках;
- `js/catalog-source.js` — источник каталога;
- `data/products.csv` — основной каталог для редактирования;
- `data/products.tsv` — резервный каталог на случай, если CSV временно недоступен.

## Как редактировать каталог

### Вариант 1 — быстро и локально

Открывайте файл `data/products.csv` и меняйте:
- название на русском и казахском;
- описание;
- цену;
- бейджи;
- изображения;
- секции каталога.

### Вариант 2 — через Google Sheets

1. Загрузите `data/products.tsv` в Google Sheets.
2. Редактируйте товары прямо в таблице.
3. Опубликуйте таблицу в интернет как CSV/TSV.
4. В файле `js/catalog-source.js` вставьте публичную ссылку в поле `googleSheetCsvUrl`.
5. После этого сайт будет брать каталог из таблицы.

Если ссылка недоступна, сайт автоматически использует локальный файл `data/products.csv`, а затем резервный `data/products.tsv`.

## Важные заметки

- Для корректной загрузки внешнего каталога сайт нужно открывать через хостинг или локальный сервер, а не напрямую через `file://`.
- Интеграция Bitrix24 и номер WhatsApp сохранены.
- Для мобильных устройств используется сценарий прямого перехода в WhatsApp, потому что обычный `window.open()` часто блокируется браузерами после async-запросов.
